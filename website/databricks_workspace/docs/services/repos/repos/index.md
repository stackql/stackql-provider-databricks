## `SELECT` examples

<Tabs
    defaultValue="get"
    values={[
        { label: 'get', value: 'get' },
        { label: 'list', value: 'list' }
    ]}
>
<TabItem value="get">

Returns the repo with the given repo ID.

```sql
SELECT
id,
head_commit_id,
branch,
path,
provider,
url
FROM databricks_workspace.repos.repos
WHERE deployment_name = '{{ deployment_name }}' -- required;
```
</TabItem>
<TabItem value="list">

Returns repos that the calling user has Manage permissions on. Use

```sql
SELECT
branch,
path,
provider,
sparse_checkout,
url
FROM databricks_workspace.repos.repos
WHERE deployment_name = '{{ deployment_name }}' -- required;
```
</TabItem>
</Tabs>


## `INSERT` examples

<Tabs
    defaultValue="create"
    values={[
        { label: 'create', value: 'create' },
        { label: 'Manifest', value: 'manifest' }
    ]}
>
<TabItem value="create">

Creates a repo in the workspace and links it to the remote Git repo specified. Note that repos created programmatically must be linked to a remote Git repo, unlike repos created in the browser.

```sql
INSERT INTO databricks_workspace.repos.repos (
data__url,
data__provider,
data__path,
data__sparse_checkout,
deployment_name
)
SELECT 
'{{ url }}',
'{{ provider }}',
'{{ path }}',
'{{ sparse_checkout }}',
'{{ deployment_name }}'
RETURNING
id,
head_commit_id,
branch,
path,
provider,
sparse_checkout,
url
;
```
</TabItem>

## `UPDATE` examples

<Tabs
    defaultValue="update"
    values={[
        { label: 'update', value: 'update' }
    ]}
>
<TabItem value="update">

Updates the repo to a different branch or tag, or updates the repo to the latest commit on the same branch.

```sql
UPDATE databricks_workspace.repos.repos
SET 
data__branch = '{{ branch }}',
data__tag = '{{ tag }}',
data__sparse_checkout = '{{ sparse_checkout }}'
WHERE 
deployment_name = '{{ deployment_name }}' --required;
```
</TabItem>
</Tabs>
